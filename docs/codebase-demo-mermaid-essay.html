Concept

Auto-generated docs aren't courses

Tools like DeepWiki and GitDiagram are great at one thing: showing you what's there. Modules, call graphs, file trees. They answer "what does this codebase contain?" in seconds.

But a course answers a different question: "what should I now believe that I didn't believe before?" That requires a promise, a route, and a payoff. Maps don't have payoffs. Essays do.

Code Walk

The schema that forces pedagogy

Every module COURSE object must declare what the reader will walk away knowing — not just what files exist. The schema makes you write it down:

references/template-essay.html → COURSE shapejs
1const COURSE = {2  module: "auth",3  learningPromise: "By the end you'll know why JWTs are stateless.",4  prereqs: ["HTTP basics", "What a cookie is"],5  diagram: "graph TD ...",  // optional anchor6  units: [7    { kind: "concept",     title, body },8    { kind: "code-walk",   title, file, code, highlightLines, explanation },9    { kind: "guess-first", question, reveal: { code, explanation } },10    { kind: "compare",     left, right, lesson },11    { kind: "surprise",    title, body },12    { kind: "takeaway",    body }13  ]14};
Lines 2–3 are the contract. If learningPromise is missing, the validator refuses to render. Line 6 onward is the only place freeform prose is allowed — and even that's typed. You can't ship a course that's just shaped like a doc.
Code Walk · Stepped

Step through renderUnit() — scroll the right column, watch the left follow

skill.js → renderUnit()js
1function renderUnit(u) {2  switch (u.kind) {3    case 'concept':4      return prose(u.body);5    case 'code-walk':6      if (u.layout === 'stepped') return steppedWalk(u);7      if (u.layout === 'split')   return splitWalk(u);8      return stackedWalk(u);9    case 'guess-first':10      return guess(u);11    case 'compare':12      return compare(u);13    case 'surprise':14      return surprise(u);15    case 'takeaway':16      return takeaway(u);17  }18}
01L1–2

Six unit kinds is the entire pedagogy surface. A flat switch is honest about that — every kind visible in one place, adding a seventh is a one-line PR.

02L6–8

Three layouts for one unit kind. Defaulting to stackedWalk means authors who don't think about layout get the right thing for free. Opt-in for the heavier modes.

03L6

Stepped is rationed. Spec says max one per module. It eats attention — you can feel it just scrolling this. Used everywhere, every page becomes a slideshow.

04L9–16

Notice the symmetry — every kind owns one renderer, no inheritance, no plugin registry. If a kind disappears, you delete one case and one renderer. That's the whole refactor.

05L18

No default: branch. The validator already rejected unknown kinds before render — by the time we get here, the schema is trustworthy. Defensive code at this layer would lie about who's responsible.

Guess First
If a module has 12 functions, how many code-walk units should the course have?

Probably 2 or 3. The instinct is "one per function" — that's what auto-doc tools do. But a course is a path, not an inventory. Pick the 2-3 functions that change the reader's mental model; mention the rest in a single concept block. The schema caps each module at 4–8 units (max 10) precisely to force this triage.

Lesson: the click pattern flatters the diagram but defeats prose. Sequence is part of the teaching.
Surprise

The Mermaid graph isn't navigation. It's a thumbnail.

The diagram at the top of this page has no click handlers. It's there so you can glance up mid-scroll and remember where you are — the same job a chapter map does in a textbook. Stripping the click behavior made the skill better, because it stopped pretending the graph was the lesson.

Concept

Cross-references stay in the prose

When one module needs to point at another, it's just a link inline: see how the parser feeds the resolver. No global flow object, no breadcrumb engine, no nav tree. Markdown links are enough — and they let voice carry the connection instead of UI chrome.

Takeaway

A codebase course earns its name when every page makes a promise and pays it off. The schema enforces the promise, the unit kinds enforce the payoff, the Mermaid graph anchors the reader, and the scroll does the rest. Auto-doc tools answer "what's here?" — this skill answers "what should you now believe?"
